# autocut-video-editing

一个基于大模型的视频自动剪辑 skill，输入源视频、音乐和提示词，输出最终成片。

## 这是什么

这个仓库不是通用应用模板，而是一个可复用的 Codex skill。

它内置了一套自动剪辑项目 `autocut`，用于把：

- 源视频
- 配乐 / 音频
- 用户提示词

组合成一条完整的自动剪辑流水线，最终生成成片视频。

## 核心原理

这套流程的核心不是“直接让模型生成视频”，而是“让模型参与视频理解、音乐理解和剪辑决策”，再由本地媒体工具完成真正的渲染。

完整思路大致分成几步：

1. 解析源视频

使用 `ffprobe` 获取时长、帧率等基础信息，再通过 `ffmpeg` 做镜头切分、抽帧和片段采样。

2. 生成候选片段

系统会根据场景变化把原视频拆成一批候选 clip，并控制最短 / 最长片段时长，避免片段过碎或过长。

3. 用视觉模型理解视频内容

对每个候选片段抽取若干帧，交给 Gemini 做视觉分析，得到片段摘要、主体、动作、氛围、剪辑价值等结构化描述。

4. 用音频模型理解配乐结构

对输入音乐做时长与段落分析，识别不同时间段的情绪、能量和适合的画面节奏。

5. 用 Agent 模型规划时间线

把用户提示词、全片上下文、候选片段分析结果和音乐段落信息综合起来，交给 OpenAI 模型决定：

- 每个音乐段落该选哪些画面
- 片段顺序怎么排
- 哪些地方更适合保留原声
- 最终节奏和情绪曲线如何推进

6. 用 ffmpeg 渲染最终成片

当时间线规划完成后，系统再调用 `ffmpeg` 完成裁切、拼接、混音和输出，生成最终视频文件。

## 模型分工

- Gemini：负责视频和音频分析
- OpenAI：负责整体规划和剪辑决策
- `ffmpeg` / `ffprobe`：负责媒体处理和最终渲染

所以它本质上是一个“多模型 + 媒体工具”的自动剪辑流水线。

## 运行方式

先在 `autocut/.env` 中填写：

- `GEMINI_API_KEY`
- `OPENAI_API_KEY`

然后在 `autocut` 下运行：

```bash
python run.py --video-path <video> --audio-path <audio> --prompt "<prompt>"
```

输出文件默认位于：

```text
autocut/output/<job_name>/
```

## 适合做什么

- 把自动剪辑项目封装成可复用 skill
- 作为 AI 视频混剪 / 自动成片流程的基础工程
- 继续扩展镜头筛选、节奏规划、原声保留、混音策略
- 接入自己的模型端点或工作流系统
